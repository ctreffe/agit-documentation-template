# Link and Reference Discipline

Links are part of the documentation structure and evidence model. They should help readers navigate, verify, and continue work.

## Internal links

Use internal links to connect related sections, procedures, concepts, decisions, and reference material.

Prefer descriptive link text over vague labels. Avoid link text such as "here" when the target can be named.

## External links

External links should be used when they provide authoritative context, current specifications, official documentation, source material, or maintained reference information.

When possible, prefer stable and authoritative sources over informal or transient pages.

## Private or unstable links

Private, internal, or unstable URLs should be documented deliberately. If they cannot be published, use a sanitized description or an internal reference model approved by the maintainer.

## Link checks

Before a milestone, check:

- Internal links resolve.
- External links are still valid.
- Link text describes the target.
- Private links are handled according to project rules.
- Links do not expose sensitive information.

## Source references

When documentation relies on source material, identify the source clearly enough that maintainers can verify it later. Do not expose sensitive source details unless approved.
