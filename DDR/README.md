# Documentation Decision Records

Documentation Decision Records (DDRs) capture important decisions about the documentation project.

Use a DDR when a decision affects:

- Documentation structure or navigation.
- Audience assumptions.
- Terminology.
- Screenshot or visual artifact policy.
- Link and source model.
- Publication workflow.
- Versioning or milestone scope.
- Handling of sensitive documentation material.

DDRs should be concise. They preserve why a decision was made, not every discussion that led to it.

## Naming

Use sequential filenames:

```text
DDR-0001-short-title.md
DDR-0002-short-title.md
```

## Template

Use [DDR-0000-template.md](DDR-0000-template.md) as the starting point for new records.
