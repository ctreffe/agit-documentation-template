# Philosophy

Technical documentation is a maintained product, not a side effect. It should be accurate, reviewable, navigable, and useful for real tasks.

## Principles

- Documentation must reflect the current technical truth.
- The maintainer owns correctness, scope, structure, and publication.
- The assistant supports structure, drafting, consistency, and QA.
- Visuals should explain real interfaces, states, workflows, or decisions.
- Links should guide readers and preserve traceability.
- Sensitive material must be protected before it enters the collaboration context.
- Decisions that shape the documentation should be recorded.
- Milestones should leave the repository in a coherent state.
